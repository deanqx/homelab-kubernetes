# Problem

## 1. On some external server

```bash
python -m http.server 8123
```

## 2. On the server

```bash
curl -i http://10.101.2.7:8123
```

Output:

```
HTTP/1.0 200 OK
Server: SimpleHTTP/0.6 Python/3.14.5
...
```

So the connection to the external server is working. The same is true from a pod
inside the cluster.

# Kubernetes Setup

Cluster consists of one server, for now called just server.

The server uses a Linux bridge but this is not the problem, I already tried it
without.

All reverse path filtering is disabled, verified by:

```bash
sudo sysctl -a | grep "rp_filter = 2"
sudo sysctl -a | grep "rp_filter = 1"
```

Both return nothing.

  name: cilium-gateway
  namespace: gateway-system
  annotations:
    cert-manager.io/cluster-issuer: lets-encrypt

  gatewayClassName: cilium

    # monitor.koiw.it works, therefore gateway routing is working

      hostname: monitor.kowi.it
